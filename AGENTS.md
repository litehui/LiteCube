# 项目组件说明

## 核心组件

### 1. Rust核心引擎

#### 1.1 功能
- **SQLite解析**：负责解析和操作SQLite文件
- **数据操作**：提供数据的增删改查功能
- **字段处理**：支持多种字段类型的处理和验证
- **API接口**：提供统一的数据操作接口

#### 1.2 技术实现
- 使用Rust语言开发
- 使用rusqlite库进行SQLite操作
- 提供FFI或WebSocket接口与前端通信
- 支持跨平台编译

### 2. Flutter移动端

#### 2.1 功能
- **用户界面**：提供移动端的用户界面
- **数据展示**：支持多种视图方式展示数据
- **数据操作**：提供数据的增删改查操作
- **与核心通信**：与Rust核心引擎进行通信

#### 2.2 技术实现
- 使用Flutter框架开发
- 使用Dart语言
- 支持iOS和Android平台
- 通过FFI或WebSocket与Rust核心引擎通信

### 3. Electron桌面端

#### 3.1 功能
- **用户界面**：提供桌面端的用户界面
- **数据展示**：支持多种视图方式展示数据
- **数据操作**：提供数据的增删改查操作
- **与核心通信**：与Rust核心引擎进行通信

#### 3.2 技术实现
- 使用Electron框架开发
- 使用Web技术栈（HTML、CSS、JavaScript）
- 支持Windows、macOS和Linux平台
- 通过Node.js的FFI模块或WebSocket与Rust核心引擎通信

## 数据流程

1. **用户操作**：用户在前端（Flutter或Electron）进行操作
2. **请求传递**：前端将请求传递给Rust核心引擎
3. **核心处理**：Rust核心引擎处理请求，操作SQLite文件
4. **结果返回**：Rust核心引擎将结果返回给前端
5. **界面更新**：前端更新界面，展示处理结果

## 开发流程

1. **核心引擎开发**：先开发Rust核心引擎，实现基本的数据操作功能
2. **前端开发**：同时开发Flutter和Electron前端应用
3. **集成测试**：测试前端与核心引擎的集成
4. **性能优化**：优化系统性能，确保流畅的用户体验
5. **功能扩展**：根据用户需求，扩展系统功能

## 技术栈

| 组件 | 技术 | 版本 | 用途 |
|------|------|------|------|
| 核心引擎 | Rust | 稳定版 | 解析SQLite文件，提供数据操作接口 |
| 移动端 | Flutter | 3.0+ | 移动端用户界面 |
| 桌面端 | Electron | 20.0+ | 桌面端用户界面 |
| 数据存储 | SQLite | 3.40+ | 数据持久化存储 |
| 通信协议 | JSON-RPC/Protocol Buffers | - | 前端与核心引擎通信 |
